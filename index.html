The 100,000 Meter View

What are Coroutines?

A way for QEMU to do
cooperative,
concurrent multitasking

Coroutines vs. Threads

  • Threads
    • OS Managed
    • Preemptive (usually)
  • Coroutines (aka Fibers)
    • Userspace Managed
    • Cooperative
    • Lower transactional overhead
What does QEMU use?

QEMU is a hybrid architecture.

Threads are used,
      in addition to coroutines (mainly in block layer)

Coroutines in QEMU

Multiple implementations:
  • ucontext
    • Default, most "mileage"
    • Not on all platforms
  • sigaltstack
    • Newer, as of 2013 (Alex Barcelo)
    • Available on most platforms
    • Not as well tested?

Coroutines in QEMU

Less common implementations:
  • win32
    • Uses Windows Fibers
    • Windows-only
  • gthreads
    • Slow
    • Not really functional
    • Debug only (anyone using it?)

Coroutines Creation - ucontext

Mixture of ucontext/makecontext and sigsetjmp/siglongjmp

  • Faster than using ucontext for all non-local gotos
  • makecontext / swapcontext used in the initial coroutine creation

Coroutines Creation - ucontext 

Coroutine *qemu_coroutine_new(void)
{

/*  [...] */

    arg.p = co;

    makecontext(&uc, (void (*)(void))coroutine_trampoline,
                2, arg.i[0], arg.i[1]);

    /* swapcontext() in, siglongjmp() back out */
    if (!sigsetjmp(old_env, 0)) {
        swapcontext(&old_uc, &uc);
    }
    return &co->base;
}
